sbox : 무작위로 섞인 단순한 난수 배열이 아님
S-box 내의 모든 256개 값(0x00 ~ 0xFF)은 "자신의 곱셈 역원을 구한 뒤, 특정 규칙으로 비트를 섞고(행렬 곱), 
마지막에 0x63을 더하는(XOR) 수학 공식"의 결과물입니다. (0x63은 AES에서 정해진 수)-(MixColumns, InvMixColumns도)
복호화에 사용되는 Inverse S-box는 이 과정의 역함수(Inverse Affine Transformation 적용 후 역원 계산)를 취하여 만들어집니다.

표준 문서의 정답(Expected CT)과 비교하는 이유
동일한 입력값(평문과 키)을 넣었을 때, 전 세계 어디서나 똑같은 암호문(출력값)이 나와야 하기때문

(Affine_Transformation)아핀 변환할때 비트를 섞는 이 방식도 공식임

V2
GF_Inverse - 기존의 브루트포스 방식 -> 페르마의 소정리를 이용한 방식 -> 확장된 유클리드 알고리즘으로 교체
MixColumns - Multiply 함수 대신 xtime을 사용하여 연산 속도와 수학적 직관성 극대화
InvMixColumns - [2, 3, 1, 1]보다 큰 [14, 11, 13, 9] 상수를 사용하여 코드가 길어지거나 헬퍼함수가 필요함

S-box 초기화 의존성 해결
외부 노출이 필요 없는 내부 함수와 배열에 static 적용
함수명에 AES_ 접두어를 붙여 네임스페이스 충돌을 방지
확장 유클리드 알고리즘에서 사용할 다항식 전용 곱셈 함수 PolynomialMultiply를 따로 구현하여 GF_Inverse 로직을 정석에 맞게 수정
매직 넘버(4)를 모두 enum 상수인 Nb로 치환
암호화 직후에도 정답(Expected CT)과 memcmp로 교차 검증하도록 테스트 로직을 보강
